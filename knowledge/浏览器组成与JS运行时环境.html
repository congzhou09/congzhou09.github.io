从浏览器到 JS 运行时环境

◆ 浏览器是多进程的，一般包括如下进程：
（1）Browser 主进程（负责浏览器基本界面的展示与交互，子页面的创建销毁与管理，网络资源的下载管理等）；
（2）渲染进程，通常说的“浏览器内核”包含在渲染进程，默认每个页面对应一个渲染进程（浏览器通常会做合并优化），负责对应页面的解析渲染与交互；
（3）GPU 进程，用于 3D 绘制等；
（4）第三方插件进程；
◆ 渲染进程是多线程的，其主要线程有：
（1）GUI 渲染线程（渲染引擎线程），负责解析 HTML 和 CSS 生成渲染树、需要 Repaint 和 Reflow 的时候进行页面内容的布局和绘制；
（2）JS 引擎线程，负责 JS 代码的解析、编译与运行；
（3）事件线程，负责控制事件循环(Event Loop)，管理任务队列(Task Queue)；
（4）定时器线程，执行 setTimeout 与 setInterval 的计数；
（5）网络线程，负责网络请求；
◆GUI 渲染线程与 JS 引擎线程是互斥的，当 JS 引擎执行时 GUI 线程被挂起，GUI 更新被保存在一个队列当 JS 引擎线程空闲时被执行；
◆ 渲染进程中的以下内容在结构上组成了“JS 运行时环境（JS Runtime Environment）”：
（1）Web API（如 setTimeout、DOM、Ajax），这些 API 由浏览器提供，在运行时环境里可访问；
（2）JS 引擎；
（3）任务队列(Task Queue)；
（4）事件循环(Event Loop)，事件循环相关规范约定于 HTML Standard；



◆JS Runtime 之于 JS 引擎的关系类似于乐曲演奏者之于乐器；
◆ 常见的 JS 引擎实例是谷歌 V8 引擎，Chrome 和 Node.js 使用了相同的 JS 引擎，但具有不同的 JS Runtime；

Event Loop

○ JS 引擎执行代码过程会涉及 JS Runtime Environment 中各个部分：
（0）JS 引擎处理的代码均来自于 Task Queue，包括 html 文档中最初的<script>标签内容；
（1）遇到变量与函数声明则存储在内存堆，遇到函数调用则将函数添加到调用栈栈顶、转而解析函数内的代码，遇到 Web API 则将函数发给 Web API；
（2）Web API 执行从调用栈发来的调用，如事件监听、Ajax 请求、定时器等，然后等待事件的触发，如点击事件、Ajax 返回、定时器时间到等，上述事件触发时将回调函数发到回调队列（Task Queue），即使当前 Call Stack 中有正在运行的代码也无须等待；
（3）Task Queue 按照收到顺序存储所有回调函数，Event Loop 不断监视 Call Stack 是否完全为空及 Task Queue 是否有内容，满足上述条件则将 Task Queue 队首的代码交给 Call Stack 执行；



○ 每一个 Event Loop 对应的 Task Queue 具体包含一个或多个宏任务(task，也称为 macrotask)队列与一个微任务(microtask，ECMAScript 中称为 job)队列，不同种类任务源的代码进入相应的队列；

○ Event Loop 不断做以下事情，一次循环称为一轮 Event Loop：
（1）从某个（具体选择策略由浏览器决定）宏队列获取队首的 task 交给执行栈执行；
（2）待 task 执行完毕，查看当前执行栈是否为空，如果为空，则不断从微队列获取队首 task 给 JS 引擎执行直至微队列清空；
（3）渲染更新，浏览器会综合“是否有 DOM 更新”及“显示帧率”等因素决定是否进行这一步；
（4）返回第一步；
○ 浏览器页面是一帧帧绘制出来的，通常帧率(FPS)为 60，即每帧时长约 16ms，每帧会完成如下 6 步任务，其中任何步骤所占时间过长导致总时间超过 16ms 都会导致帧率下降进而导致视觉上的卡顿，每帧内做的事情如下：
（1）处理用户的交互
（2）JS 的解析与执行
（3）帧开始，处理窗口尺寸变更，页面滚动等
（4）rAF(requestAnimationFrame)
（5）布局
（6）绘制



○ requestIdleCallback(callback[, options])用于在帧间空闲阶段执行 callback 内的低优先级代码，options 可取属性为 timeout，表示如果 timeout 时间内 callbakc 还未被执行则必然会在下个帧间阶段执行，即使由此可能导致卡顿；

同步与异步、阻塞与非阻塞

■ 结合上述 JS 运行时环境运行 JS 代码的两个特性：同步与异步、阻塞与非阻塞
（1）同步和阻塞指的是 Call Stack 里的执行过程是单线程，同时只能执行栈顶函数，栈顶函数执行完毕并从 Call Stack 推出之前，Call Stack 里的其他函数将被阻塞；
（2）异步和非阻塞指的是调用浏览器提供的 Web API 时不会占用 Call Stack，Call Stack 后面的代码可以继续执行，只有 Web API 返回的回调函数要执行的时候才回到 Call Stack 里；

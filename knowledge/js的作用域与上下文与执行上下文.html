作用域

○ 作用域(scope)用来约定变量的可访问特性，它将代码划分成一个个区域。根据所在位置，作用域可分为全局作用域和局部作用域（如函数作用域、块级作用域）。
○ 块级作用域通过在{}里使用 let 或 const 声明变量实现。
○ 函数作用域通过函数体划分。
○ 全局作用域是不在任何函数作用域或块级作用域中的区域。

作用域链

● 由于函数和块可以嵌套，使作用域也可以存在上下层嵌套关系，上下层关系具有称为“作用域链(scope chain)”的机制。作用域链有如下特性：
（1）在每个局部作用域内可以向上访问上层局部作用域直到全局作用域中的变量，但不能向下访问嵌套函数或嵌套块内的变量。
（2）若在局部作用域使用一个变量在局部不存在，则不断向上层查找直至找到或在全局作用域创建（非严格模式且找不到时）。
（3）若在局部作用域声明一个与上层变量同名的变量，则在局部作用域内变量值为作用域内的值，而在上级作用域同名变量仍是上级作用域内的值。

执行上下文

□ 执行上下文(execution context)是 JS 代码执行涉及的状态信息的统称。
□ 根据代码段的位置，可将执行上下文可分为三种类型：（1）全局执行上下文，在所有代码还未执行前创建（2）函数执行上下文，函数调用时创建（3）eval 执行上下文，执行 eval 语句时创建。
□没有块级执行上下文，，执行块级代码时候仍然会在当前的执行上下文进行。
□ 所有的执行上下文存储在执行上下文栈(execution context stack)，随着代码从全局入口执行到调用函数到函数返回，执行上下文也再不断被创建、入栈、切换、出栈、销毁，当前正在执行的代码段对应的执行上下文永远在栈顶。此过程可以借助代码执行可视化工具直观了解。
□ 每个执行上下文可分为创建阶段(creation Phase)与执行阶段(execution Phase)，执行阶段才真正执行代码。
□ 创建阶段会做如下事情：
（1）代码段区域中声明的变量，为其分配空间，如果是 var 声明则赋值 undefined，否则不赋值。
（2）代码段区域中声明的函数，将其载入内存。
（3）变量和函数被记录在当前执行上下文的词法环境(lexical environment)，词法环境中还有个“外部词法环境”通常取值为上层词法环境的引用，由此实现作用域链的建立。
□ 创建阶段对声明变量与函数的处理实现了“声明提前(hoisting)”。let 和 const 声明的变量也实现了声明提前，但不能提前使用。

上下文

■ 上下文(context)通常说的就是 this 。
■ this 存在于所有的执行上下文中，取值规则如下：
（1）在全局执行上下文，this 值指向全局对象（浏览器环境就是 window ）。
（2）在函数执行上下文，this 值指向调用当前函数的对象。
（3）eval 执行上下文的 this 值还与是 direct eval 还是 indirect eval有关，对于 direct eval，this 值与 eval 语句所在执行上下文的 this 一致；对于 indirect eval，等同于 eval 语句在全局执行上下文，此时 this 值指向全局对象。

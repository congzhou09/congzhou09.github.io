JS运行时环境(JS Runtime Environment)

2019-11-05
  • 知识解惑

概述

◆浏览器中都具有JS运行时环境(JS Runtime Environment)，而运行时环境里包含了：
（1）Web API，如setTimeout、DOM、Ajax，这些API由浏览器提供，在运行时环境里可访问；
（2）JS引擎，用于代码的解析与编译；
（3）回调队列(Callback Queue)；
（4）事件循环(Event Loop)；



◆JS Runtime之于JS引擎的关系类似于乐曲演奏者之于乐器；
◆常见的JS引擎实例是谷歌V8引擎，Chrome和Node.js使用了相同的JS引擎，但具有不同的JS Runtime；

JS引擎

○JS引擎负责JS的解析和JIT编译成机器码，主要包含如上图黑框中的两部分：内存堆（Memory Heap，内存分配的地方）、调用栈（Call Stack，随着代码运行堆栈帧所在地方）；
○JS引擎处理代码过程：
（1）部分解析代码以检查语法错误；
（2）从上到下解析与编译所有代码成机器码；

Web API & Callback Queue & Event Loop

○JS引擎解析代码过程涉及了JS Runtime Environment中各个部分：
（1）遇到变量与函数声明则存储在内存堆，遇到函数调用则将函数添加到调用栈栈顶、转而解析函数内的代码，遇到Web API则将函数发给Web API；
（2）Web API执行从调用栈发来的调用，如事件监听、Ajax请求、定时器等，然后等待事件的触发，如点击事件、Ajax返回、定时器时间到等，上述事件触发时将回调函数发到回调队列（Callback Queue），即使当前Call Stack中有正在运行的代码也无须等待；
（3）Callback Queue按照收到顺序存储所有回调函数，Event Loop不断监视Call Stack是否完全为空及Callback Queue是否有内容，满足上述条件则通知Callback Queue将队首的回调函数发到Call Stack执行；



同步与异步、阻塞与非阻塞

■结合上述JS运行时环境解释JS语言的两个特性：同步与异步、阻塞与非阻塞
（1）同步和阻塞指的是Call Stack里的执行过程是单线程，同时只能执行栈顶函数，栈顶函数执行完毕并从Call Stack推出之前，Call Stack里的其他函数将被阻塞；
（2）异步和非阻塞指的是调用浏览器提供的Web API时不会占用Call Stack，Call Stack后面的代码可以继续执行，只有Web API返回的回调函数要执行的时候才回到Call Stack里；
